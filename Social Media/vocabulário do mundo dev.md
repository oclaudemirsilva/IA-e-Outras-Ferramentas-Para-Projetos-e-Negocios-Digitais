

## Lock-in  
Lock-in em desenvolvimento (dev lock-in) é quando um desenvolvedor ou empresa fica "preso" a uma tecnologia, ferramenta ou fornecedor específico, tornando extremamente difícil ou caro migrar para uma solução alternativa devido a custos de mudança elevados, formatos proprietários, falta de portabilidade de dados ou conhecimento especializado. Isso acontece quando a solução inicial é conveniente, mas cria dependência, limitando a liberdade de escolha e dificultando a troca para concorrentes, sendo comum em plataformas low-code, nuvem (Cloud Vendor Lock-in) e ecossistemas fechados.

Exemplos Comuns
Tecnologias Proprietárias: Usar formatos de arquivo ou APIs que não funcionam bem com outras ferramentas.
Plataformas de Baixo Código (Low-Code): Aplicativos desenvolvidos em uma plataforma podem não ser facilmente portáveis para outra.
Cloud (Nuvem): Dificuldade em mover dados e aplicações de um provedor de nuvem para outro.
Sistemas Legados: A empresa continua usando um software antigo porque o custo de migrar ou treinar a equipe para uma nova solução é muito alto. 
Como Evitar o Dev Lock-in
Use Padrões Abertos: Adote tecnologias e protocolos abertos como REST, JSON, SQL.
Arquitetura Flexível: Use containers (Docker) e Infraestrutura como Código (IaC) para facilitar a portabilidade.
Estratégias Multi-cloud: Distribua cargas de trabalho entre vários provedores para ter poder de barganha.
Planeje a Saída (Exit Strategy): Simule a migração de dados e sistemas desde o início.
Mantenha os Dados Portáveis: Garanta backups e formatos de dados que permitam a migração.
